Quick Start: Submodules and Build

Fresh checkout:

git clone https://github.com/QiangWu769/msquic_cellular.git
cd msquic_cellular
git submodule update --init --recursive

If the repository is already checked out, refresh the submodules with:

cd /home/qwu26/msquic_cellular
git submodule sync --recursive
git submodule update --init --recursive

Build the secnetperf test binary:

cd /home/qwu26/msquic_cellular
rm -rf build
cmake -S . -B build \
  -DCMAKE_BUILD_TYPE=Release \
  -DQUIC_BUILD_PERF=ON \
  -DQUIC_ENHANCED_PACKET_LOGGING=ON \
  -DQUIC_TLS_LIB=openssl
cmake --build build --target secnetperf -j"$(nproc)"

The expected output binary is:

/home/qwu26/msquic_cellular/build/bin/Release/secnetperf

Build and push cellninjia_mobile from the repository root:

cd /home/qwu26/msquic_cellular
make -C tools/cellninjia/cellninjia_mobile clean all
make -C tools/cellninjia/cellninjia_mobile push

make -C tools/cellninjia/cellninjia_mobile all cross-compiles cellninjia_mobile for Android ARM64 with the Android NDK. On the Buffalo Ubuntu host, the Makefile defaults to:

/home/qwu26/android-ndk-r26d

Override it if needed:

make -C tools/cellninjia/cellninjia_mobile clean all ANDROID_NDK_HOME=/path/to/android-ndk

make -C tools/cellninjia/cellninjia_mobile push installs the binary on the connected phone as:

/data/local/tmp/cellninjia_mobile

Forward the bridge port and run the bridge on the phone:

adb forward tcp:43555 tcp:43555
adb shell su -c /data/local/tmp/cellninjia_mobile

Keep cellninjia_mobile running while the Python DIAG parsers in tools/cellninjia/ connect to 127.0.0.1:43555.

A minimal GBR-CC upload run looks like:

./build/bin/Release/secnetperf \
  -target:<server-ip> \
  -port:4433 \
  -cc:bbr \
  -upload:20mb \
  -ptput:1 \
  -cellular:1

Run logs are written under:

/home/qwu26/msquic_cellular/artifacts/bbr_logs/

The legacy bbr_logs path is kept as a local compatibility symlink to artifacts/bbr_logs.

Project Overview

This repository is the MsQuic/secnetperf transport prototype for the paper "Synchronizing with the Scheduler: Dual-Loop Congestion Control for 5G Uplink on Commodity Devices".

The implementation adds a cellular scheduler-facing signal, the Grant-to-BSR Ratio (GBR), to the QUIC upload path. A DIAG/CellNinjia parser computes GBR from BSR and uplink grant/PUSCH telemetry, sends it over a Unix datagram socket, and MsQuic uses it to drive a ratio-based pacing and congestion-window controller.

Main Components

  • src/core/bbr.c: MsQuic BBR hook point and GBR-driven pacing/cwnd logic.
  • src/perf/lib/cellular_ratio.c: Unix socket receiver for GBR samples.
  • src/perf/lib/SecNetPerfMain.cpp: secnetperf integration and -cellular command-line mode.
  • tools/cellninjia/: DIAG/CellNinjia parsers for LTE and 5G BSR/grant extraction.
  • tools/build-scripts/: convenience build wrappers.
  • analysis/transport/: transport-log analysis scripts.
  • docs/gbr-cc/: repository layout and GBR-CC notes.
  • artifacts/bbr_logs/: local experiment logs and generated plots. This directory is kept out of Git and should be archived separately.

Important Modes

  • -cc:bbr: enables the BBR congestion-control implementation used as the host for GBR-CC.
  • -cellular:0: receives and logs GBR samples without changing congestion-control behavior.
  • -cellular:1: enables GBR-driven control of pacing rate and cwnd.
